# 🏨 Booking Application

Асинхронное веб-приложение на FastAPI для бронирования номеров в отелях. Проект спроектирован с учетом современных практик разработки, поддерживает фоновые задачи, кэширование данных и контейнеризацию.

## ✨ Ключевые возможности (Features)
- 🔐 **Аутентификация и авторизация:** Безопасная система пользователей на базе JWT-токенов.(Используется библиотека bcrypt в многопоточном режиме, тк она блокирует целый поток на подсчет хэша)
- 📅 **Управление бронированием:** Поиск, создание, проверка статусов и отмена броней номеров.
- ⚡ **Кэширование:** Интеграция с Redis для ускорения частых запросов к API.
- 📊 **Фоновые задачи:** Использование Celery для асинхронной обработки задач (например, генерация отчетов, проверка статуса оплаты).
- 🧪 **Тестирование:** Полное покрытие интеграционными и юнит-тестами с использованием изолированной тестовой БД.

---

## 🛠️ Технологический стек (Tech Stack)

- **Backend Framework:** FastAPI (Python 3.11+)
- **ORM / Database:** SQLAlchemy 2.0 (Async) + PostgreSQL
- **Migrations:** Alembic
- **Task Queue:** Celery + Redis
- **Testing:** Pytest (pytest-asyncio)
- **Environment:** Pydantic v2 (Settings)

## 🚀 Быстрый старт (Getting Started)

### Требования (Prerequisites)
Убедитесь, что у вас установлены:
- Python 3.11 или выше
- PostgreSQL и Redis (либо Docker)

### 1. Клонирование репозитория
```bash
git clone [https://github.com/Ularik/booking.git)

2. Настройка переменных окружения
Создайте файл .env в корневой директории проекта. Вы можете скопировать структуру из примера ниже:

Фрагмент кода

MODE=  # TEST/ # LOCAL/ # DOCKER

DB_NAME=
DB_HOST=
DB_HOST_DOCKER=
DB_USER=
DB_PASSWORD=
DB_PORT=

REDIS_HOST=
REDIS_HOST_DOCKER=
REDIS_PORT=6379

SECRET_KEY=
ALGORITHM=
ACCESS_TOKEN_EXPIRE_MINUTES=
3. Локальный запуск (через виртуальное окружение)
Bash
# Создание и активация виртуального окружения
python -m venv .venv
source .venv/bin/activate  # Для Linux/macOS
.venv\Scripts\activate     # Для Windows

# Установка зависимостей
pip install -r req.txt

# Создайте Базу postgresql, Например: booking
# Применение миграций базы данных
alembic upgrade head

# Запуск основного приложения FastAPI
uvicorn src.main:app --port 8000 --reload
4. Запуск Celery Воркера
Для обработки фоновых задач запустите воркер в отдельном терминале (с активированным виртуальным окружением):

Bash
celery -A src.tasks.tasks:celery_app worker --loglevel=info
(Примечание: для Windows может потребоваться добавить флаг -P gevent или -P solo)

🧪 Запуск тестов
В проекте настроены автоматические тесты, которые используют изолированную базу данных. Перед запуском убедитесь, что тестовая БД создана.

Выполните команду в корне проекта:

Bash
pytest -v -s
